# BIO 라벨링 가이드

## 1. 목적

본 문서는 식품 성분표 텍스트에서 알레르기 관련 성분을 탐지하기 위한 KoBERT NER 학습 데이터의 BIO 라벨링 기준을 정의한다.

초기 분류 대상은 다음 5개 알레르기 항목으로 제한한다.

* 우유: MILK
* 계란: EGG
* 땅콩: PEANUT
* 대두: SOY
* 밀: WHEAT

분류 대상이 아닌 성분은 모두 `O`로 라벨링한다.

---

## 2. BIO 라벨 목록

| 라벨       | 의미                       |
| -------- | ------------------------ |
| O        | 알레르기 관련 성분이 아닌 일반 토큰     |
| B-MILK   | 우유 관련 성분의 시작             |
| I-MILK   | 여러 토큰으로 구성된 우유 관련 성분의 내부 |
| B-EGG    | 계란 관련 성분의 시작             |
| I-EGG    | 여러 토큰으로 구성된 계란 관련 성분의 내부 |
| B-PEANUT | 땅콩 관련 성분의 시작             |
| I-PEANUT | 여러 토큰으로 구성된 땅콩 관련 성분의 내부 |
| B-SOY    | 대두 관련 성분의 시작             |
| I-SOY    | 여러 토큰으로 구성된 대두 관련 성분의 내부 |
| B-WHEAT  | 밀 관련 성분의 시작              |
| I-WHEAT  | 여러 토큰으로 구성된 밀 관련 성분의 내부  |

## 3. 기본 라벨링 원칙

### 3.1 한 토큰으로 이루어진 성분

성분이 하나의 토큰으로 구성된 경우 `B-클래스`만 사용한다.

예시:

| 토큰   | 라벨       |
| ---- | -------- |
| 전지분유 | B-MILK   |
| 난백액  | B-EGG    |
| 땅콩분태 | B-PEANUT |
| 대두유  | B-SOY    |
| 밀가루  | B-WHEAT  |
| 설탕   | O        |

### 3.2 여러 토큰으로 나누어진 성분

하나의 성분이 두 개 이상의 토큰으로 분리된 경우 첫 번째 토큰에는 `B-`, 이후 토큰에는 `I-`를 사용한다.

예시:

| 토큰  | 라벨     |
| --- | ------ |
| 유청  | B-MILK |
| 단백질 | I-MILK |

단, `유청단백질`처럼 붙어 있는 경우에는 하나의 토큰으로 보고 `B-MILK`를 부여한다.

### 3.3 분류 대상이 아닌 알레르기 성분

현재 분류 대상이 아닌 복숭아, 호두, 새우, 게, 쇠고기, 아황산류 등의 성분은 우선 `O`로 처리한다.

예시:

| 토큰   | 라벨 |
| ---- | -- |
| 복숭아  | O  |
| 새우   | O  |
| 아황산류 | O  |

향후 클래스를 확장할 경우 별도의 BIO 라벨을 추가한다.

### 3.4 ‘함유’와 일반 문구

`함유`, `원재료명`, `제품`, `제조시설` 등의 일반 단어에는 `O`를 부여한다.

예시:

| 토큰 | 라벨      |
| -- | ------- |
| 밀  | B-WHEAT |
| 우유 | B-MILK  |
| 함유 | O       |

## 4. 라벨링 예시

### 예시 1

문장:

밀가루, 설탕, 전지분유, 대두유

토큰:

밀가루 | 설탕 | 전지분유 | 대두유

라벨:

B-WHEAT | O | B-MILK | B-SOY

### 예시 2

문장:

땅콩분태, 물엿, 난백액, 버터

토큰:

땅콩분태 | 물엿 | 난백액 | 버터

라벨:

B-PEANUT | O | B-EGG | B-MILK

### 예시 3

문장:

밀, 달걀, 우유, 대두 함유

토큰:

밀 | 달걀 | 우유 | 대두 | 함유

라벨:

B-WHEAT | B-EGG | B-MILK | B-SOY | O

### 예시 4

문장:

이 제품은 땅콩을 사용한 제품과 같은 제조시설에서 제조하였습니다.

토큰:

이 | 제품은 | 땅콩을 | 사용한 | 제품과 | 같은 | 제조시설에서 | 제조하였습니다

라벨:

O | O | B-PEANUT | O | O | O | O | O

조사까지 붙은 `땅콩을`과 같은 토큰을 그대로 사용할지, `땅콩`과 `을`로 분리할지는 모델 담당자와 협의하여 하나의 기준으로 통일한다.

## 5. 데이터 작성 규칙

* `tokens`와 `labels`의 개수는 반드시 같아야 한다.
* 토큰 구분자는 `|`를 사용한다.
* 라벨 구분자도 `|`를 사용한다.
* 원본 OCR 결과는 별도 열에 보관한다.
* 사람이 이미지를 보고 교정한 텍스트를 `corrected_text`에 저장한다.
* 판독할 수 없는 이미지는 학습 데이터에서 제외한다.
* 애매한 성분을 추측하여 라벨링하지 않는다.
* 같은 성분은 모든 데이터에서 동일한 라벨을 사용한다.

## 7. 검수 기준

데이터 저장 전 다음 항목을 확인한다.

1. 이미지 파일명과 텍스트가 일치하는가?
2. `corrected_text`가 실제 이미지 내용과 일치하는가?
3. `tokens`와 `labels`의 개수가 같은가?
4. 성분 사전에 등록된 클래스와 라벨이 일치하는가?
5. 분류 대상이 아닌 성분은 `O`로 처리했는가?
6. 불확실한 성분을 임의로 추측하지 않았는가?
